# User Question Policy

## 목적

에이전트를 설계하는 데 필요한 정보가 부족할 때 다음 중 적절한 행동을 선택한다.

1. 현재 요청에서 확인한다.
2. 기존 대화에서 확인한다.
3. 프로젝트나 자료를 탐색한다.
4. 합리적인 추론을 사용한다.
5. 안전한 기본값을 사용한다.
6. 사용자에게 질문한다.
7. 행동 권한이 필요한 경우 승인을 요청한다.

질문은 정보 수집의 첫 번째 수단이 아니라 마지막 수단에 가깝다.

---

## 질문 유형

### 요구사항 확인 질문

목적, 범위 또는 출력이 불명확할 때 사용한다.

예시:

> 이 에이전트는 UI 설계안만 생성하나요, 아니면 실제 프로젝트 파일도 수정하나요?

### 선택 질문

여러 선택지가 가능하고 사용자 선호가 결과에 큰 영향을 줄 때 사용한다.

### 규약 확인 질문

필수 규약이 있다고 알려졌지만 실제 내용을 확인할 수 없을 때 사용한다.

### 승인 요청

위험하거나 외부 영향이 있는 작업을 수행할 권한을 얻기 위한 요청이다.

요구사항 질문과 승인 요청은 구분한다.

---

## 질문 전 해결 순서

```text
현재 요청
→ 기존 대화
→ 프로젝트와 문서 탐색
→ 기존 구현 패턴
→ 합리적인 추론
→ 안전한 기본값
→ 사용자 질문
```

---

## 반드시 질문해야 하는 조건

다음 조건에 해당하고 탐색이나 기본값으로 해결할 수 없다면 질문한다.

- 답에 따라 에이전트 수나 스킬 구조가 크게 달라진다.
- 설계형인지 실행형인지 구분할 수 없다.
- 작업 대상 프로젝트나 시스템을 특정할 수 없다.
- 필수 프로젝트 또는 조직 규약에 접근할 수 없다.
- 요구사항이 서로 충돌한다.
- 성공 기준을 일반적인 기준으로도 정의할 수 없다.
- 사용자 선호 자체가 핵심 결과물이다.
- 안전, 비용, 외부 영향 또는 되돌리기 어려운 결정에 영향을 준다.

---

## 질문하지 않아야 하는 조건

다음은 일반적으로 질문하지 않는다.

- 에이전트 이름
- 스킬 폴더명
- 문서 제목
- 출력 섹션 순서
- Markdown 사용 여부
- 프로젝트 파일에서 확인 가능한 기술 스택
- 프로젝트 설정에서 확인 가능한 테스트 도구
- 쉽게 변경할 수 있는 낮은 영향도의 설정
- 일반적인 안전 기본값이 존재하는 결정

---

## 판단 기준

### 영향도

- 낮음: 이름, 문서 순서, 설명 형식
- 중간: 출력 구조, 선택 스킬
- 높음: 책임 범위, 권한, 기술 환경, 안전 정책

### 복구 가능성

- 쉽게 복구: 이름, 설명 순서
- 복구 비용 있음: 스킬 인터페이스, 폴더 구조
- 복구 어려움: 데이터 모델, 운영 자동화, 권한 모델

### 탐색 가능성

프로젝트에서 확인할 수 있다면 먼저 탐색한다.

### 외부 영향

이메일 발송, 배포, 데이터 변경, 결제 등은 승인 정책과 연결한다.

### 정보 신뢰도

`UNRESOLVED + 높은 영향도 + 탐색 불가 + 복구 어려움`일 때 질문 가능성이 가장 높다.

---

## 질문 작성 원칙

- 질문 이유를 설명한다.
- 한 번에 관련 질문을 묶는다.
- 선택지를 제공할 수 있으면 제공한다.
- 권장 기본값을 함께 제시한다.
- 이미 답한 내용을 다시 묻지 않는다.
- “자세히 알려주세요”처럼 범위가 불분명한 질문을 피한다.

좋은 질문:

> 실제 파일을 수정할 수 있는 에이전트인지에 따라 도구와 승인 정책이 달라집니다. 분석과 제안만 수행하게 할까요, 아니면 일반적인 소스 파일 생성·수정까지 허용할까요? 기존 프로젝트용 개발 에이전트라면 소스 수정까지 허용하고 패키지 설치와 배포는 별도 승인으로 두는 구성을 권장합니다.

---

## 하위 스킬의 질문 정책

하위 스킬은 기본적으로 사용자에게 직접 질문하지 않는다.

대신 다음과 같은 상태를 반환한다.

```yaml
status: NEEDS_CLARIFICATION
reason:
  type: missing_mandatory_rule
  description: 사내 디자인 규약에 접근할 수 없음
affected_outputs:
  - ui_architecture
suggested_question:
  적용해야 할 디자인 시스템 문서나 핵심 규칙을 제공해 주세요.
```

최종 질문 여부는 상위 에이전트가 판단한다.
